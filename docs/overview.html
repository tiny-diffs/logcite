1.Commands

Three subcommands. compress is the one log-reading command — --stats and --templates are alternate views of the same pass. Input is a <file> or - (stdin); JSON output goes to stdout (or --output).

commandinputoutput
compress <file|->raw log linesan IncidentCapsule (compact JSON)
compress … --statsraw log linesnumbers only: lines, tokens, compression, timing
compress … --templatesraw log linesthe template breakdown (id, count, share, level, pattern)
expand <file>a seekable file + --lineraw lines around a cited line (window from..to)
validate <file|->a capsule JSON{ valid, schema, errors[] }

To compress a live source, pipe into it — no wrapper command needed: kubectl logs -n prod api 2>&1 | logpod compress -

4.Scoring & causal roles

Each candidate line gets a 0..1 score; severity dominates, rarity and numeric spikes break ties.

# anomaly.ts
score = min(1, 0.65·severity + 0.30·rarity + spike)
  severity  TRACE/DEBUG 0 · INFO 0.05 · WARN 0.45 · ERROR 0.85 · FATAL 1
  rarity    min(1, log(total / templateCount) / log(total))   # IDF-like
  spike     +0.20 if the line's max number ≥ p95 of numeric maxima

Roles are assigned relative to the strongest error (the root_cause), gated to a window around it so unrelated earlier errors aren't mislabeled.

pre-window (timestamps)120 s
post-window (timestamps)600 s
pre-window (no ts)200 lines
post-window (no ts)5000 lines
triggerpre & (WARN or score≥0.5)
root_causestrongest error, earliest tie
consequencepost & hint/ERROR/WARN
contexteverything else kept

5.Redaction, levels & timestamps

ANSI color/escape codes are stripped first (so pino-pretty and colored stdout parse cleanly). Then PII is masked before templating (default on; disable with --no-redact):

patternreplaced with
email addresses<email>
IPv4 addresses<ip>
UUIDs<uuid>
bearer / JWT-like tokens<token>

Timestamps detected at the start of a line (or inline for access logs), parsed to epoch ms:

formatexample
ISO-86012026-05-04T14:22:11.123Z
Rails bracketed ISOI, [2026-05-04T14:22:11.123 #4321]
slash (nginx / Go std log)2026/05/04 14:22:11
epoch seconds / millis1717689600 · 1717689600123
RFC3164 syslog (year assumed)May 4 14:22:11
CLF access log (inline)[10/Oct/2000:13:55:36 -0700]
time-only bracketed (pino-pretty)[12:34:56.789]  (date = today)
JSONL ts fieldsts / timestamp / time / date (epoch s or ms, or string)

Severity level is read from the text (TRACE…FATAL, case-insensitive) or, for JSON, from these fields — including numeric scales:

sourceaccepted
JSON fieldslevel · severity · status · lvl · PRIORITY
numeric — pino/bunyan10 trace · 20 debug · 30 info · 40 warn · 50 error · 60 fatal
numeric — syslog PRIORITY0–2 fatal · 3 error · 4 warn · 5–6 info · 7 debug
fallbackif no level field, the message text is scanned (e.g. CloudWatch events)

Out-of-range numbers (e.g. an HTTP status of 200/500) are not treated as a level.

6.Capsule schema logpod.incident_capsule/v1

{
  "schema": "logpod.incident_capsule/v1",
  "service": string,
  "window": "HH:MM:SS to HH:MM:SS" | "unknown",
  "compression": number   // round(tokens_in / tokens_out),
  "stats": {
    "lines_in": number,
    "tokens_in_est": number   // sampled estimate (o200k_base),
    "tokens_out_est": number   // exact
  },
  "evidence": [{
    "role": "root_cause" | "trigger" | "consequence" | "context",
    "line": number   // 1-based source line (the citation),
    "text": string   // redacted,
    "template": string   // e.g. "T5",
    "score": number   // 0..1
  }],
  "routine_summary": {
    "total_lines": number,
    "template_count": number,
    "top_templates": [{ "id": string, "pattern": string, "count": number }]
  }
}

7.Exit codes & streams

codemeaning
0ok — capsule produced / capsule valid
1input — missing file or parse problem
2usage — unknown flag or command
3schema — capsule failed validation

JSON → stdout (or --output). Diagnostics and timing → stderr.
